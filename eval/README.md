# Eval set for `swmf-improve`

This directory holds the train + holdout papers the meta-agent
`swmf-improve` uses to grow the `rules/` layer without contaminating it
from its own gold answers.

## Layout

```
eval/
├── README.md                       (this file)
├── expert_queue.md                 operator-knowledge backlog
├── papers/
│   ├── holdout.txt                 names of held-out papers
│   └── <paper-name>/
│       ├── paper.pdf               the paper itself (user-provided)
│       ├── reference/
│       │   └── PARAM.in            gold answer (user-provided)
│       ├── cache/                  agent-generated paper_spec.json etc.
│       └── runs/
│           └── <YYYY-MM-DD>/       one directory per `/swmf-improve` cycle
│               ├── attempt/        replication output (PARAM, REPORT, CONFIG, job)
│               ├── score.md        agent's A–E score against the rubric
│               ├── gap_list.md     categorized gaps
│               ├── improvement_plan.md
│               └── proposals/      rule proposals for this cycle
└── proposals/
    ├── <YYYY-MM-DD>-<gap>.md       human-review-pending proposals
    └── rejected/                   proposals rejected by the provenance check
```

## What to provide (user)

Only two files per paper:

- `papers/<name>/paper.pdf` — the paper itself
- `papers/<name>/reference/PARAM.in` — the gold answer, **physically
  isolated** from any `swmf_root` the agent operates in

Everything else is generated by the agent.

## What is forbidden

- Do not link to anything under `eval/papers/*/reference/` from rule
  files in `src/agent_assets/skills/support/swmf-params/rules/`. The
  meta-agent's provenance check will reject any rule whose cited source
  resolves under a reference directory.
- Do not author `paper_spec.json` by hand for an eval paper. The agent
  generates it from `paper.pdf` and caches it under `cache/`.

## Holdout discipline

See `papers/holdout.txt`. Briefly: a paper exits holdout after the
agent scores composite-clean on it twice in a row, then joins train and
a new paper enters holdout. This rotation is what keeps the eval signal
honest as the rules layer grows.
